# Проект: Подготовка доклада "Системные баги под нагрузкой" для Heisenbug

## Контекст

Готовим доклад для конференции **Heisenbug** от имени **Михаила Жилина** — инженера по производительности с 15+ летним стажем (Postgres Professional). Доклад на **40 минут** рассказа.

**Тема:** Расследование бага в ядре Linux (XFS + init_on_free + memory pressure = segfault), длившееся год.

**Ключевое сообщение:** Когда система падает "без причин" — причина есть всегда, и чтобы её найти, нужно идти вглубь стека: от приложения через ОС до ядра.

## Структура проекта

### Профиль спикера и правила
- `speaker-profile.md` — полный профиль: tone of voice, скорость речи, любимые фразы, метафоры, стиль юмора
- `talk-guidelines.md` — структура доклада, правила DO/DON'T, примеры текста голосом Михаила
- `vocabulary.md` — компактный словарь: филлеры, уменьшительные, метафоры, антисловарь

### Исходные материалы
- `texts/*.srt` — расшифровки двух выступлений Михаила (tone of voice reference)
- `texts/article.md` — статья на Хабре "История поиска бага в ядре Linux" (основа доклада)
- `texts/article-comments.md` — 22 комментария, сгруппированы по 5 темам для Q&A
- `texts/article-references.md` — 3 ссылки на коммиты ядра Linux, хронология, глоссарий
- `articles/` — MHTML с lore.kernel.org + детальный Markdown-разбор серии из 18 патчей

### Доклад (готовые материалы)
- `talk/draft.md` — полный текст доклада (~3900 слов, 9 разделов, голос Михаила)
- `talk/slides.md` — описание 31 слайда с визуалами
- `talk/qa-prep.md` — 10 ожидаемых вопросов из зала с ответами

## Правила работы с текстом

### Голос спикера
- Весь текст — **голосом Михаила Жилина**: разговорный, неформально-технический, с самоиронией
- Слова-связки: "вот", "ну", "то есть", "блин", "мы такие", "вот смотрите"
- Уменьшительные: "табличка", "базенка", "виртуалочка", "ссылочка"
- "Мы", а не "я" — история команды
- Технические термины: термин + "то есть" + простое объяснение

### Структура доклада (9 разделов)
1. Вступление — зацепка (3 мин)
2. Завязка — симптомы у клиентов (4 мин)
3. Уровни стека — фреймворк мышления (4 мин)
4. Первые зацепки — нули вместо кода (5 мин)
5. Три условия — XFS + init_on_free + memory pressure (6 мин)
6. Воспроизведение — конкретные тесты (5 мин)
7. Root cause — race condition, один автор, два коммита (5 мин)
8. Выводы — алгоритм расследования, командная работа (4 мин)
9. Финал (1 мин)

### Чего избегать
- Академического стиля и канцеляризмов
- Абстрактных рассуждений без примеров
- Мата (максимум "блин")
- Шуток на чей-то счет
- Агрессивного маркетинга (польза для компании должна быть органичной)
