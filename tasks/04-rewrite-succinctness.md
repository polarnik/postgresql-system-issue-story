# Шаг 4: Переписать для сжатости

## Принцип Shapiro

> "Succinctness is a ratio — the percentage of significant thoughts out of all thoughts communicated."
> "Readers often quit not because they dislike your ideas, but because they're bored."

Для устного доклада: слушатели отвлекаются на телефон, если 30 секунд подряд нет ничего интересного.

## 3-шаговый процесс Shapiro

### Step 1: Переписать разделы по памяти

Для каждого раздела доклада:
1. Прочитать раздел
2. Закрыть файл
3. Записать раздел заново, только самое важное
4. Сравнить: что отвалилось — это было fluff

**Альтернатива (verbal summary):** Попросить кого-то прочитать раздел и пересказать за 30 секунд. То, что осталось — ядро.

### Step 2: Убрать лишние слова

В каждом предложении спросить:
- "Зачем я это сказал? Какой эффект на слушателя?"
- "Есть ли что-то более полезное, что можно сказать вместо этого?"

Убрать:
- Повторы идей (если уже сказал — не говори снова)
- Описания, которые не нужно описывать
- Слова, не добавляющие контекста

### Step 3: Перефразировать абзацы с нуля

Зная, что хочешь сказать → переписать максимально коротко.

## Тест "Tweet test" (адаптированный для доклада)

Можешь ли сжать каждый раздел в одно предложение?

| Раздел | Tweet |
|--------|-------|
| 1. Вступление | "Postgres падал с сегфолтом без причин, и мы целый год искали почему" |
| 2. Завязка | "Клиенты жаловались, дампы показывали не на Postgres, и мы решили копать" |
| 3. Стек | "Проблема может быть на любом уровне — от приложения до ядра ОС" |
| 4. Зацепки | "В памяти нули вместо кода; не только Postgres; больше нагрузки = чаще падает" |
| 5. Три условия | "XFS + init_on_free + memory pressure = сегфолт, по отдельности безобидны" |
| 6. Воспроизведение | "Воспроизвели на виртуалке за 5 минут: mmap + malloc-пожиратель → нули в данных" |
| 7. Root cause | "Race condition в ядре: один автор сломал (2022) и починил (2024) через 2 года" |
| 8. Выводы | "6 шагов расследования + командная работа + смотри глубже своего уровня" |

Если tweet пустой или неинтересный — раздел нужно пересмотреть.

## Конкретные места для сжатия (гипотезы)

- [ ] Раздел 3 (Стек): возможно, слишком долго объясняет очевидное для аудитории Heisenbug
- [ ] Раздел 5 (Три условия): возможно, mmap объяснён избыточно
- [ ] Раздел 6 (Воспроизведение): bash-команды можно сократить до слайда
- [ ] Раздел 8 (Выводы): проверить, нет ли повторов из предыдущих разделов

## Цитата-напоминание

> "If it is a ten minute speech, it takes me all of two weeks to prepare it. If it is a half-hour speech, it takes me a week. If I can talk as long as I want to, it requires no preparation at all." — The Operative Miller

40 минут — это не "сколько угодно". Каждая минута должна оправдывать себя.

## Критерий завершения

- [ ] Каждый раздел сжимаем в tweet без потери сути
- [ ] Нет абзацев, которые повторяют уже сказанное
- [ ] Нет "вводных" предложений, которые не несут информации
- [ ] Тайминг по разделам проверен (не больше заявленного)
