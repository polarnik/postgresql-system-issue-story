# Шаг 03: Rewrite для ясности — что сделано

## Главные изменения

### Раздел 3 (Стек) — радикально переписан
**Было:** Абстрактная лекция про уровни стека, аналогия с врачом, цитата Грегга, рекомендации аудитории.
**Стало:** История спуска команды по уровням стека ("залезли в логи Postgres — ничего, пошли в ОС — нашли нули, пошли в ФС — только XFS"). Стек показывается ЧЕРЕЗ историю, а не отдельно.
**Добавлена аналогия:** "Как археология — копаешь первый слой, черепки; второй — фундамент; третий — целый город."
**Убрано:** Цитата Грегга (отвлекала), менторские рекомендации, "самое полезное из доклада" (спойлер).
**Сокращено:** С ~4 мин до ~3 мин. Высвободилось время для других разделов.

### Раздел 5 (Три условия) — три правки ясности
1. **XFS:** Убрано прямое обращение к аудитории Heisenbug. Объяснение стало нейтральным.
2. **mmap:** Полностью переписан. Было: абстрактное "механизм отображения файлов". Стало: конкретная последовательность — ОС не копирует файл, она его отображает → reclaim выгружает страницу → init_on_free обнуляет → программа получает нули. Причинно-следственная цепочка стала явной.
3. **reclaim:** Объяснён по-русски ("ОС ищет, что выгрузить из памяти, чтобы освободить место").

### Раздел 7 (Race condition) — упрощён kernel-жаргон
**Было:** kcompactd, deferred split queue, folio_put, folio_undo_large_rmappable, list_del_init — стена терминов.
**Стало:** "Структура данных", "общая очередь", "локальный список", "указатели". Суть сохранена, жаргон убран.
**Добавлена аналогия:** "Два человека разбирают цепочку звеньев с разных концов — и цепочка рвётся."

### Мелкие правки
- "Сегфолт" объяснён при первом упоминании в Р3 (segmentation fault — обращение к чужой памяти)
- "OOM Killer" объяснён в Р6 (механизм ядра, убивающий процессы при нехватке памяти)
