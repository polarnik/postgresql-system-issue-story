## Слайд 14: Условие 1 — XFS
- **Заголовок:** "Файловая система XFS"
- **Почему энтерпрайз:** блоки до 64 КБ, 8-КБ страницы Postgres, журналирование, многопоточность
- **Факт:** "Дефолт в энтерпрайзе для больших БД"
- **Визуал:** Логотип XFS, может сравнение XFS vs ext4

---

## Слайд 15: Условие 2 — init_on_free
- **Заголовок:** "Обнуление памяти при освобождении"
- **Что это:** Параметр ядра Linux с версии 5.3
- **Зачем:** Безопасность — чтобы следующий процесс не прочитал чужие данные
- **Ирония:** "Параметр для безопасности → обнуляет код"
- **Как проверить:** `mem auto-init: heap free:on` в логах
- **Визуал:** Замок/щит → нули

---

## Слайд 16: Условие 3 — Давление на память
- **Заголовок:** "Memory pressure"
- **Что это:** Ядро освобождает кэшированные страницы (reclaim)
- **mmap:** Базовая функциональность с 1990-х, используется для загрузки ЛЮБОГО бинарника
- **Визуал:** Схема: файл на диске → mmap → страница памяти → reclaim → обнуление → 💀

## Слайд 24: Root cause — коммит-преступник
- **Заголовок:** "Один человек, два коммита, два года"
- **Коммит:** `56a4d67` — "mm/readahead: Switch to page_cache_ra_order"
- **Автор:** Matthew Wilcox (Oracle)
- **Дата:** Март 2022
- **Масштаб:** 3 файла, +5/-5 строк
- **Подпись:** "Шесть строчек кода — два года боли"

---

## Слайд 25: Race condition — диаграмма
- **Визуал:** Sequence diagram с двумя CPU:
  - CPU 1: берёт folio из deferred queue → перемещает в локальный список → отпускает блокировку → итерирует список
  - CPU 2: финальный folio_put → берёт блокировку на deferred queue → list_del_init на folio, которое в локальном списке CPU 1
  - 💥 Список CPU 1 повреждён
- **Подпись:** "Классический race condition. Стреляет только под нагрузкой"

---

## Слайд 26: Исправление
- **Коммит:** `bc2ff4c` — "mm: free folios in a batch in shrink_folio_list()"
- **Автор:** Тот же Matthew Wilcox (Oracle)
- **Дата:** Март 2024
- **Суть:** linked list → folio_batch (массив из 15 элементов)
- **Бонус:** массив итерируется в 5-15x быстрее
- **Визуал:** linked list (перечёркнут) → массив
